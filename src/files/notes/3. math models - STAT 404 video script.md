## Functions as models 
- Explicit and implicit relationships between variables

An explicit model of something is simply a function that describes its behavior, and it does so explicitly. A model of exponential growth like that of bacteria (but with base e instead of base 2) can be written as 

$y = Ne^{\theta t}$

y is the output itself of the function, t is the dependent variable denoting time (instead of  x, but it can really be any variable you like), N is the initial population size and \theta governs the timeframe at which population increases - the two last ones are parameters in the model.

For example, if time is given in days and for some made up values like $N=10, \theta = 1/2$, we can compute the function value at some points:

i. at time t=0 zero the exponential term is one (e^0 = 1 for any base) so the population size described by this model is simply N (making clear this parameter is indeed the initial population size); 
ii. at time t=1 we have Ne^(1/2) (i.e. the initial population size multiplying the square root of the Napierian number, which is greater than one) 
iii. at time t=2 we have y=Ne

and so on...

- Implicit models
More often than not, we do not know what is the function that describes a system. Instead, we may have some knowledge about the system in the form of individual processes. Fortunately, this kind of knowledge can be put together into implicit models, which may take the form of differential equations. In our population reproduction example, suppose we do not know its growth is exponential, but only that population grows proportionally to its size; we can formalize that (verbally, initially) in the language of rates of change: the rate of growth in time of population size is proportional to the current size of the population. Mathematically, we write:

dy/dt = \theta y

the parameter theta is included because although growth is proportional to population size, the relationship may not be one-to-one. In this case however, we have not made an explicit statement about the population size, but about its growth. We can, however, generate an explicit output from this implicit description.

## Not all models are useful, but all are mathematical

- Anything can be a mathematical model
Think of any equation, anything, it could be a mathematical model, if your data looks like some function, you could image that usign that function to describe the relationship between input and output could be helpful. The question really becomes how is it helpful: if you need to understand why the data looks like it does, maybe you need more than a function that resembles the data, it may need tro explain why the data looks like that; if you need a model that is very close to all data you see, and will probably look like new data, it may need to be something more flexible.
        
- Mechanistic models: all models are wrong, but some are mechanistic
As we mentioned, we can build some models from known processes, when you are trying to really reconstruct (to some extent) how the system works, what you are building si a mechanistic model. We saw a few examples, and complexity can escalate quickly for anything but the simples systems.
